## 1. Resumen ejecutivo

- **Total hallazgos:** 6 (Críticos: 1, Altos: 2, Medios: 2, Bajos: 0, Info: 1)
- **Top 3 riesgos:**
  1. SSRF en `scraper-api` por validación insuficiente de URL (CWE-918; OWASP A10:2021) (OWASP Foundation, 2021).
  2. Endpoint de scraping sin autenticación/rate limit: abuso + DoS + amplifica SSRF.
  3. Migrations Supabase inconsistentes (Clerk JWT vs `auth.uid()`): riesgo de RLS roto/mal aplicado.
- **Recomendación inmediata:** bloquear SSRF (denylist/allowlist de destinos + bloquear IPs privadas/metadata), restringir exposición del endpoint (auth o red privada) y estabilizar las migrations/policies de Supabase antes de despliegues.

## 4. Hallazgos

### F-001 — SSRF: `scraper-api` navega URLs arbitrarias con validación insuficiente

- **Severidad:** Crítica
- **CVSS v3.1:** 9.6 (AV:N/AC:L/PR:N/UI:N/S:C/C:H/I:L/A:H)
- **Categoría:** OWASP A10:2021 SSRF / CWE-918
- **Estado:** Confirmado
- **Ubicación:** `scraper-api/internal/audit/handler.go:24-26,59`
- **Descripción:** la validación `validURL()` solo verifica `http/https` y no bloquea IPs privadas, `localhost`, rangos link-local (ej. 169.254.169.254) ni hosts internos. Luego se ejecuta `page.Goto(req.URL, ...)`, permitiendo que el servidor haga requests a destinos arbitrarios.
- **Impacto:** acceso a recursos internos/no expuestos (metadata cloud, paneles internos), escaneo de red, y potencial pivot a otros servicios (OWASP Foundation, 2021; OWASP Cheat Sheet Series, s. f.).
- **PoC conceptual:**
  - `POST /api/audit {"url":"http://169.254.169.254/latest/meta-data/"}` (cloud metadata)
  - `POST /api/audit {"url":"http://localhost:2375/"}` (servicios locales si existieran)
- **Remediación sugerida:**
  - Resolver DNS y bloquear IPs privadas (RFC1918), loopback, link-local, multicast, y rangos reservados.
  - Permitir solo `https` (opcionalmente `http` en dev), bloquear puertos no estándar o mantener allowlist de puertos.
  - (Recomendado) agregar un “egress policy” a nivel red (VPC/SG/firewall) y/o ejecutar el scraper en red sin acceso a infraestructura interna.
- **Esfuerzo estimado:** M
- **Referencias (si aplica):** (OWASP Foundation, 2021); (OWASP Cheat Sheet Series, s. f.)

### F-002 — Endpoint de scraping sin autenticación: abuso + amplifica SSRF/DoS

- **Severidad:** Alta
- **CVSS v3.1:** 8.6 (AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:H)
- **Categoría:** OWASP A01:2021 Broken Access Control / OWASP A05:2021 Security Misconfiguration
- **Estado:** Confirmado
- **Ubicación:** `scraper-api/cmd/server/main.go:31`
- **Descripción:** `POST /api/audit` no requiere auth (API key, mTLS, JWT, allowlist de caller) y dispara cómputo costoso (Playwright + crawl). CORS no reemplaza auth.
- **Impacto:** terceros pueden consumir recursos (DoS económico/CPU), usar el servicio como proxy de crawling y explotar SSRF.
- **PoC conceptual:** flood de requests concurrentes a `/api/audit` desde múltiples IPs/domains.
- **Remediación sugerida:**
  - Exigir autenticación (API key compartida, HMAC, JWT) o restringir por red (private network / firewall allowlist) y agregar rate limiting.
- **Esfuerzo estimado:** M
- **Referencias (si aplica):** (OWASP Foundation, 2021)

### F-003 — CORS permisivo por defecto (`ALLOWED_ORIGIN=*`) y uso potencial como “control”

- **Severidad:** Alta
- **CVSS v3.1:** 6.5 (AV:N/AC:L/PR:N/UI:R/S:U/C:L/I:N/A:H)
- **Categoría:** OWASP A05:2021 Security Misconfiguration
- **Estado:** Confirmado
- **Ubicación:** `scraper-api/internal/config/config.go:18`, `scraper-api/cmd/server/main.go:26`
- **Descripción:** el valor por defecto permite cualquier origin. Además, aunque se configure un origin, CORS solo aplica a navegadores: no bloquea requests server-to-server ni herramientas.
- **Impacto:** cualquier sitio web puede invocar el endpoint desde el browser de un usuario (y leer la respuesta si el origin coincide con `*`), facilitando abuso distribuido.
- **PoC conceptual:** página web maliciosa que hace `fetch("https://<scraper>/api/audit", {method:"POST", body:...})`.
- **Remediación sugerida:**
  - En prod: `ALLOWED_ORIGIN` = URL exacta del dashboard (y evitar `*`).
  - No confiar en CORS como auth: combinar con auth/rate limiting o aislamiento de red.
- **Esfuerzo estimado:** S
- **Referencias (si aplica):** (MDN Web Docs, s. f.)

### F-004 — Riesgo de DoS por falta de rate limiting / timeouts integrales

- **Severidad:** Media
- **CVSS v3.1:** 6.5 (AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H)
- **Categoría:** OWASP A05:2021 Security Misconfiguration
- **Estado:** Probable
- **Ubicación:** `scraper-api/cmd/server/main.go`, `scraper-api/internal/audit/handler.go:59`, `scraper-api/internal/audit/site_tree.go:89`
- **Descripción:** el endpoint ejecuta operaciones costosas (browser automation + crawl). No se observa rate limiting, quotas por IP, ni circuit breaker. Aunque hay timeouts puntuales (Goto 25s, crawl 2s/request), un atacante puede saturar el pool.
- **Impacto:** degradación del servicio, colas, timeouts y caída.
- **PoC conceptual:** múltiples requests simultáneos con URLs lentas.
- **Remediación sugerida:** rate limiting (por IP/API key), límites de concurrencia por caller, timeouts por request globales, y budget de crawl configurable.
- **Esfuerzo estimado:** M
- **Referencias (si aplica):** (OWASP Foundation, 2021)

### F-005 — Migrations Supabase inconsistentes (Clerk JWT vs Supabase Auth) con riesgo de RLS incorrecto

- **Severidad:** Media
- **CVSS v3.1:** 5.3 (AV:N/AC:L/PR:L/UI:N/S:U/C:L/I:L/A:N)
- **Categoría:** OWASP A05:2021 Security Misconfiguration / OWASP A01:2021 Broken Access Control
- **Estado:** Probable
- **Ubicación:**
  - Clerk-style (texto + `auth.jwt()->>sub`): `dashboard-web/supabase/migrations/0001_clerk_rls_foundation.sql:14,58,92,129`
  - Supabase-auth-style (`auth.uid()` + FK a `auth.users`): `dashboard-web/supabase/migrations/0002_supabase_auth_profiles.sql:14,55-63` y `dashboard-web/supabase/migrations/0003_seo_snapshots.sql:18-25`
- **Descripción:** el repo contiene migraciones que mezclan dos modelos de identidad (Clerk third-party JWT vs Supabase auth.users). En entornos donde se apliquen todas las migraciones, existe riesgo de policies que no compilen por mismatch de tipos (`uuid` vs `text`) o que reemplacen policies con condiciones incorrectas.
- **Impacto:** RLS roto (denegación total) o, peor, políticas mal definidas que expongan datos cruzados.
- **PoC conceptual:** aplicación de migrations en un proyecto nuevo y verificación de policies/errores; o chequeo de RLS efectivo con JWT de Clerk.
- **Remediación sugerida:**
  - Consolidar a **un solo** set de migrations compatibles con Clerk.
  - Archivar/eliminar las migrations legacy o convertirlas en no-op seguro.
  - Agregar validación en CI (apply migrations en DB efímera + smoke de RLS).
- **Esfuerzo estimado:** M
- **Referencias (si aplica):** (Supabase, s. f.)

### F-006 — Dependencia vulnerable reportada por `npm audit`: PostCSS XSS (transitiva)

- **Severidad:** Info
- **CVSS v3.1:** 3.1 (AV:N/AC:H/PR:N/UI:R/S:U/C:L/I:N/A:N)
- **Categoría:** OWASP A06:2021 Vulnerable and Outdated Components
- **Estado:** Probable
- **Ubicación:** `dashboard-web` (dependencia transitiva vía `next` según `npm audit --omit=dev`)
- **Descripción:** `npm audit` reporta vulnerabilidad moderada en `postcss <8.5.10` por posible XSS en salida de stringify de CSS (GitHub, s. f.).
- **Impacto:** depende de si el proyecto serializa CSS no confiable; típicamente afecta toolchains/build. Riesgo contextual.
- **PoC conceptual:** N/A (riesgo depende del uso de stringify con input no confiable).
- **Remediación sugerida:** evaluar upgrade alineado de Next/PostCSS (sin force-downgrade) y monitorear advisory.
- **Esfuerzo estimado:** S
- **Referencias (si aplica):** (GitHub, s. f.)
